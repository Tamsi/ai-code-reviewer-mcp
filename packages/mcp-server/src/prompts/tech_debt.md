# Task: Technical debt analysis

Identify technical debt and design smells. Focus on:

- Code duplication and copy-paste that should be factored out.
- High coupling, low cohesion, and leaky abstractions.
- Long functions/classes, deep nesting, and high cyclomatic complexity.
- Dead code, unused exports, and commented-out blocks.
- Inconsistent patterns and abstractions across the codebase.
- TODO/FIXME/HACK markers and workarounds that should be addressed.
- Missing or weak module boundaries.

For each item, describe the maintenance cost it imposes and a refactoring direction.
